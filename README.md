# Учет инсталляционных комплектов (InstallKits)

Программа для учета инсталляционных комплектов (ИК), состоящих из **носителей** (дисков) и **коробок**.

Доступны два интерфейса:
- **Десктопное приложение** — CustomTkinter (Windows, Linux)
- **Веб-приложение** — Flask + Bootstrap (работа через браузер)

Оба интерфейса используют общий слой данных (`data/`) и одну базу TinyDB.

## Возможности

- **Комплектация** — создание продуктов с составом (носители и коробки)
- **Остатки** — приход и корректировка компонентов на складе
- **Списание** — пакетное списание комплектов по таблице или списание с произвольной датой
- **Просмотр** — расчет доступного количества комплектов по каждому продукту
- **Отчеты** — формирование и экспорт в Excel (.xlsx)
- **История операций** — журнал всех изменений
- **Настройки** — путь к базе данных (локальный файл или сетевой UNC-путь)

## Требования

- Python **3.12** или **3.13** (не 3.14+)
- [Poetry](https://python-poetry.org/) для управления зависимостями

## Установка и запуск

### Установка зависимостей

```bash
cd InstallKits
poetry install
```

Локальное виртуальное окружение создается в папке `.venv` (настроено в `poetry.toml`).

### Десктопное приложение

```bash
poetry run python main.py
```

Или с активацией окружения:

```powershell
# Windows
.\.venv\Scripts\Activate.ps1
python main.py
```

```bash
# Linux
source .venv/bin/activate
python main.py
```

### Веб-приложение

```bash
poetry run python webapp/app.py
```

Приложение будет доступно по адресу: **http://localhost:5000**

Подробнее — в [webapp/README.md](webapp/README.md).

## Списание комплектов

### Веб-версия (страница «Списание»)

1. В таблице укажите количество в колонке **«Списать»** для нужных продуктов
2. Нажмите **«Списать»** — списываются все строки, где количество > 0 (текущая дата)
3. **«Списать ИК»** — списание одного продукта с выбранной датой

### Десктопная версия

- Кнопка **«Списать ИК»** в боковой панели — списание одного комплекта с указанием даты

## Настройки

Файл `settings.json` в корне проекта:

```json
{
  "db_path": "data/db.json",
  "window_geometries": { ... }
}
```

- `db_path` — путь к файлу базы данных (локальный или сетевой, например `//server/share/db_IK.json`)
- `window_geometries` — размеры и позиции окон (только десктоп)

Путь к БД можно изменить на вкладке **«Настройки»** в десктопном приложении.

## Сборка исполняемого файла (Windows)

Подробная инструкция — в [BUILD.md](BUILD.md).

```bash
poetry run pyinstaller installkits.spec
```

Готовый файл: `dist/InstallKits.exe`

## Сборка для Linux

```bash
./package_linux.sh
```

Инструкция по созданию RPM-пакета — в [RPM_PACKAGING.md](RPM_PACKAGING.md).

## Структура проекта

```
InstallKits/
├── main.py                 # Точка входа (десктоп)
├── settings_manager.py     # Управление настройками
├── settings.json           # Настройки приложения
├── pyproject.toml          # Зависимости Poetry
├── InstallKits.spec        # Конфигурация PyInstaller
├── BUILD.md                # Сборка .exe
├── RPM_PACKAGING.md        # Сборка RPM
├── package_linux.sh        # Сборка Linux-архива
│
├── data/                   # Общий слой данных
│   ├── db.py               # Работа с TinyDB
│   ├── products.py         # Продукты (комплекты)
│   ├── discs.py            # Носители
│   ├── boxes.py            # Коробки
│   ├── kits.py             # Состав комплектов
│   ├── stock.py            # Остатки и списание
│   └── operations.py       # История операций
│
├── ui/                     # Десктопный интерфейс
│   ├── main_window.py      # Главное окно
│   ├── report_window.py    # Окно отчетов
│   ├── tabs/               # Вкладки
│   │   ├── products_tab.py
│   │   ├── stock_tab.py
│   │   ├── view_stock_tab.py
│   │   └── settings_tab.py
│   └── dialogs/            # Диалоговые окна
│
└── webapp/                 # Веб-интерфейс
    ├── app.py              # Flask-приложение
    ├── templates/          # HTML-шаблоны
    └── static/             # CSS, JavaScript
```
