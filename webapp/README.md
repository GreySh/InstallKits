# Учет ИК — Веб-приложение

Веб-интерфейс для системы учета инсталляционных комплектов. Использует общий слой данных из папки `data/` и настройки из `settings.json`.

## Запуск

### Рекомендуемый способ (через Poetry из корня проекта)

```bash
cd InstallKits
poetry install
poetry run python webapp/app.py
```

### Альтернативный способ (локальное окружение)

```powershell
# Windows
.\.venv\Scripts\Activate.ps1
python webapp\app.py
```

```bash
# Linux
source .venv/bin/activate
python webapp/app.py
```

Приложение будет доступно по адресу: **http://localhost:5000**

## Разделы приложения

| Раздел | URL | Описание |
|--------|-----|----------|
| Главная | `/` | Обзор продуктов и статистика |
| Списание | `/view-stock` | Остатки комплектов, пакетное списание |
| Остатки | `/stock` | Приход и корректировка носителей/коробок |
| Комплектация | `/kits` | Управление продуктами и составом |
| Отчеты | `/reports` | Остатки, уровни, экспорт в Excel |
| Операции | `/operations` | История всех операций |

## Основные операции

### Добавить остаток

1. Перейдите в раздел **«Остатки»**
2. Выберите вкладку **«Носители»** или **«Коробки»**
3. Выберите компонент и укажите количество
4. Нажмите **«Добавить»**

### Списать комплекты (пакетно)

1. Перейдите в раздел **«Списание»**
2. В таблице укажите количество в колонке **«Списать»** для нужных продуктов
3. Нажмите **«Списать»** — списываются все строки, где количество > 0

Перед списанием отображается подтверждение со списком продуктов.

### Списать с другой датой

1. На странице **«Списание»** нажмите **«Списать ИК»**
2. Укажите дату, продукт и количество
3. Нажмите **«Списать»**

### Создать продукт

1. Перейдите в раздел **«Комплектация»**
2. Нажмите **«Добавить продукт»**
3. Укажите название и состав (носители/коробки с количеством)

## API

### Данные

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/products` | Все продукты |
| GET | `/api/discs` | Все носители |
| GET | `/api/boxes` | Все коробки |
| GET | `/api/stock` | Все остатки |
| GET | `/api/stock/disc/<id>` | Остаток носителя |
| GET | `/api/stock/box/<id>` | Остаток коробки |
| GET | `/api/operations` | История операций |

### Списание

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/dispatch` | Списание одного продукта (с опциональной датой) |
| POST | `/dispatch-batch` | Пакетное списание (JSON: `[{product_id, quantity}, ...]`) |

### Экспорт

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/export-stock-to-excel` | Экспорт остатков в Excel |
| GET | `/reports/operations/export` | Экспорт операций в Excel |
| GET | `/reports/stock-levels/export` | Экспорт уровней остатков |

## Настройка

### Порт и хост

По умолчанию: `host='0.0.0.0'`, `port=5000`.

Для изменения отредактируйте конец файла `app.py`:

```python
app.run(debug=False, host='0.0.0.0', port=8080)
```

### База данных

Путь к базе данных задается в `settings.json` в корне проекта:

```json
{
  "db_path": "data/db.json"
}
```

Для работы нескольких пользователей можно указать сетевой путь:

```json
{
  "db_path": "//server/share/db_IK.json"
}
```

### Режим отладки

Для продакшена установите `debug=False` в `app.run()`.

## Безопасность

Приложение предназначено для **локального или корпоративного использования** в доверенной сети.

Для публичного развертывания рекомендуется:
1. Использовать HTTPS (через reverse proxy)
2. Добавить аутентификацию
3. Отключить режим отладки
4. Вынести `secret_key` в переменные окружения

## Технологии

- **Backend:** Python 3.12+, Flask 3
- **Frontend:** HTML5, Bootstrap 5.3, JavaScript
- **База данных:** TinyDB (общая с десктопной версией)

## Устранение неполадок

1. Проверьте логи в консоли, где запущен сервер
2. Убедитесь, что зависимости установлены: `poetry install`
3. Проверьте доступность пути к БД в `settings.json`
4. При ошибке `ModuleNotFoundError` — запускайте через `poetry run` или активируйте `.venv`
