# План разработки: Учет инсталляционных комплектов

## Описание проекта
Программа учета инсталляционных комплектов с графическим интерфейсом на базе CustomTkinter и TinyDB.

## Архитектура

### Структура базы данных (TinyDB)

**Таблицы:**
1. **`products`** - виды продуктов, с указанием комплектации (id, name, components)
2. **`discs`** - виды дисков (id, name) - name уникален
3. **`boxes`** - виды коробок (id, name) - name уникален
4. **`kit_components`** - состав комплектов (id, product_id, disc_id, box_id, disc_quantity, box_quantity)
5. **`stock_discs`** - остатки дисков (id, disc_id, quantity)
6. **`stock_boxes`** - остатки коробок (id, box_id, quantity)
7. **`operations`** - история операций (id, operation_type, product_id, quantity, date, details)

### Слой данных (data/)
- `db.py` - инициализация базы данных, получение таблиц
- `products.py` - CRUD для продуктов
- `discs.py` - CRUD для дисков
- `boxes.py` - CRUD для коробок
- `kits.py` - управление составом комплектов
- `stock.py` - управление остатками (корректировка)
- `operations.py` - запись и получение истории

### Слой представления (ui/)
- `main_window.py` - главное окно с вкладками для разделов
- `tabs/` - вкладки для разделов:
  - `products_tab.py` - управление продуктами
  - `stock_tab.py` - управление остатками
  - `view_stock_tab.py` - просмотр остатков
- `dialogs/` - диалоговые окна:
  - `add_product.py` - добавление нового комплекта
  - `add_disc.py` - добавление нового вида диска
  - `add_box.py` - добавление нового вида коробки
  - `add_all.py` - добавление дисков/коробок (существующих)
  - `adjust_stock.py` - корректировка остатков
  - `dispatch.py` - списание комплекта
  - `delete_disc.py` - удаление диска
  - `delete_box.py` - удаление коробки
- `report_window.py` - формирование отчета и экспорт в .xlsx

## Функциональные модули

### 1. Добавление нового инстал комплекта
- Форма: название продукта
- Выбор состава: диски и их количество, коробки и их количество
- Создание в `products` и `kit_components`

### 2. Добавление дисков/коробок
- Форма: название, количество
- Проверка уникальности названия
- Добавление в `stock_discs` или `stock_boxes`

### 3. Корректировка остатков
- Выбор вида диска/коробки
- Форма: новое количество (положительное число)
- Обновление остатков

### 4. Удаление дисков и коробок
- Выбор диска/коробки из списка
- Подтверждение удаления
- Удаление из базы данных и обновление остатков

### 5. Списание инстал комплекта
- Форма: дата отгрузки (по умолчанию сегодня), название продукта, количество
- Проверка доступных остатков компонентов
- Уменьшение остатков компонентов
- Запись в историю

### 5. Просмотр остатков
- **По продуктам**: название, доступное количество комплектов (минимальное из доступных компонентов)
- **По дискам**: название, остаток
- **По коробкам**: название, остаток

### 6. Отчет по остаткам
- Экспорт в файлы .xlsx (Excel) через openpyxl
- Структура: продукт, количество, дата отчета

## Исправления и улучшения

### Исправление отображения текущего остатка в диалоге "Корректировка"
- Проблема: При выборе элемента в диалоге корректировки не отображался текущий остаток
- Причина: CTkComboBox не поддерживает событие `<<ComboboxSelected>>` и `_variable` создаётся с задержкой
- Решение:
  - Использован `ttk.Combobox` вместо `CTkComboBox` для списка элементов (как рекомендовано в constraint-ах)
  - `ttk.Combobox` корректно поддерживает событие `<<ComboboxSelected>>`
  - Обработчик `on_item_change` теперь правильно получает и отображает текущий остаток
- Проблема: Неправильная передача ссылок на вкладки
- Решение:
  - Исправлены методы `adjust_stock()`, `add_all()` в `StockTab` для передачи правильных ссылок на вкладки
  - Исправлены диалоги `AdjustStockDialog`, `AddAllDialog`, `DispatchDialog` для корректного получения ссылок на вкладки
  - Добавлена логика: если параметры переданы, они используются, иначе получаем из главного окна

### Исправление обновления вкладок при добавлении новых дисков/коробок
- Проблема: Вкладка "Просмотр" не обновлялась при добавлении новых дисков или коробок
- Решение:
  - Обновлены `AddDiscDialog` и `AddBoxDialog` для принятия параметров `stock_tab` и `view_tab`
  - Обновление вкладок добавлено в метод `ok()` диалогов после успешного добавления
  - В `StockTab` методы `add_disc()` и `add_box()` теперь передают правильные ссылки на вкладки
- Проблема: Вкладки "Остатки" и "Просмотр" не обновлялись после операций корректировки и списания
- Решение:
  - Исправлены методы `adjust_stock()`, `add_all()` в `StockTab` для передачи правильных ссылок на вкладки
  - Исправлены диалоги `AdjustStockDialog`, `AddAllDialog`, `DispatchDialog` для корректного получения ссылок на вкладки через `nametowidget('.')`
  - Добавлена логика: если параметры переданы, они используются, иначе получаем из главного окна

### Исправление получения ссылок на вкладки
- Проблема: Неправильное получение ссылок на вкладки через `self.master.master`
- Решение: Использование `master.nametowidget('.')` для получения корневого виджета (MainWindow)

## Известные проблемы и ограничения

### CustomTkinter ограничения
- CustomTkinter не имеет встроенного TreeView - используется `ttk.Treeview` из стандартного tkinter
- CustomTkinter не имеет встроенного MessageBox - используется `tkinter.messagebox`
- CustomTkinter не имеет CTkToplevel - используется `tkinter.Toplevel` для диалогов
- CTkComboBox по умолчанию показывает "CTkComboBox" если не задано значение - используется `set("")`
- При использовании CTkComboBox с кириллицей рекомендуется использовать `type_index` вместо сравнения строк

### TinyDB
- При использовании `CachingMiddleware` данные не сохраняются - используется обычный `TinyDB` без кэширования

### Структура виджетов
- `self.master` у `CTkScrollableFrame` - это фрейм вкладки, а не `MainWindow`
- Для получения `MainWindow` используется `master.nametowidget('.')`
